## 二、新发现问题

### 2.1 P1遗留：`init_memory_db.py` schema字段名未改

**位置**：`scripts/init_memory_db.py` 第31行

```python
vector_summary TEXT,   # ← 应改为 text_summary
```

**影响**：
- 该脚本创建的数据库表使用旧字段名`vector_summary`
- 若其他代码（`__init__.py`、`database.py`）使用`text_summary`，则存在字段名不一致风险
- 新用户执行初始化后将创建错误的表结构

**建议**：将`CREATE TABLE memories`中的`vector_summary`改为`text_summary`

---

### 2.2 P1遗留：TEST.md Test#1仍引用旧字段名

**位置**：`references/TEST.md` 第19行（表格第1行）

```
| 1 | 记忆库写入 | reply+vector_summary不为空 |
```

**影响**：测试用例与新字段名`text_summary`不一致，测试将失败

**建议**：改为`reply+text_summary不为空`

---

### 2.3 P2遗留：版本号未更新为V1.1

**位置**：`SKILL.md` 文件头

```markdown
**版本**：V1.0   ← 应更新为 V1.1
**更新日期**：2026-03-29
```

**Changelog章节**仅有V1.0条目，无V1.1修复记录。

**建议**：版本改为V1.1，Changelog添加V1.1行，记录2个P0+2个P1的修复内容。
